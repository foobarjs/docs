# Mail

foobarjs provides a mail abstraction with multiple drivers for sending emails.

## Configuration

Configure mail in `config/mail.js`:

```js
export default {
  driver: process.env.MAIL_DRIVER || 'log',
  from: process.env.MAIL_FROM || 'hello@example.com',
  smtp: {
    host: process.env.MAIL_HOST || 'localhost',
    port: parseInt(process.env.MAIL_PORT || '1025'),
  },
}
```

When the framework boots, it reads this config and calls `Mailer.configure()` automatically. You can also configure manually:

```js
Mailer.configure({ driver: 'array' })
```

## Available Drivers

| Driver | Description |
|--------|-------------|
| `log` | Writes emails to `storage/logs/mail.log` (default) |
| `array` | Stores emails in memory (for testing) |
| `smtp` | Sends via SMTP using emailjs |

## Basic Usage

```js
import { Mailer } from 'foobarjs'

await Mailer
  .to('user@example.com')
  .subject('Order Confirmation')
  .html('<h1>Thanks for your order!</h1>')
  .send()
```

No `new` needed — `Mailer` is a singleton that uses the config from `config/mail.js`.

## Fluent API

### `to(address)`

Set the recipient. Accepts a string or array of strings:

```js
Mailer.to('alice@example.com')
Mailer.to(['alice@example.com', 'bob@example.com'])
```

### `subject(line)`

Set the email subject:

```js
Mailer.subject('Welcome to Foobar')
```

### `html(content)`

Set the HTML body:

```js
Mailer.html('<h1>Hello</h1><p>Welcome to our platform.</p>')
```

### `text(content)`

Set the plain text body. If omitted, it's auto-generated by stripping HTML tags:

```js
Mailer.text('Hello, welcome to our platform.')
```

### `from(address)`

Override the default sender address:

```js
Mailer.from('noreply@example.com')
```

### `send()`

Send the email. Returns the send result with a `sentAt` timestamp:

```js
const result = await Mailer.send()
// { to: [...], subject, html, text, from, sentAt }
```

### Configuring per-message

All static methods create a new instance internally, so you can also pass config per-chain:

```js
const mail = new Mailer({ driver: 'smtp', ... })
await mail.to('user@example.com').subject('Hi').send()
```

## Testing

Use the `array` driver to keep emails in memory for test assertions:

```js
import { Mailer } from 'foobarjs/mail'

Mailer.configure({ driver: 'array' })
await Mailer.to('test@example.com').subject('Test').text('Hello').send()

const sent = Mailer.getArrayDriverMessages()
assert.strictEqual(sent.length, 1)
assert.strictEqual(sent[0].subject, 'Test')
```

Clear between tests:

```js
Mailer.clearArrayDriver()
```

### Test Mock Helper

The `mock.mail.sent()` helper from `foobarjs/test` also inspects the array driver:

```js
test('sends welcome email', async ({ mock }) => {
  // ... code that sends email using array driver ...
  const emails = await mock.mail.sent()
  assert.ok(emails.length > 0)
})
```

## SMTP Driver

For production, use the `smtp` driver:

```js
const mail = new Mailer({
  driver: 'smtp',
  from: 'noreply@example.com',
  smtp: {
    host: 'smtp.sendgrid.net',
    port: 587,
    username: 'apikey',
    password: process.env.SENDGRID_API_KEY,
  },
})
```

## Log Driver

The `log` driver writes each email to `storage/logs/mail.log`:

```
--- Mail ---
From: foobar@localhost
To: user@example.com
Subject: Order Confirmation
Time: 2026-01-01T12:00:00.000Z
Text: Thanks for your order!
HTML: <h1>Thanks for your order!</h1>
--- End ---
```
