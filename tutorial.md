# Build your first foobarjs app

You'll build **links-app** — a tiny link manager that stores titles and URLs
and counts clicks. By the end you'll have a home page, a detail page, a
create form, a working admin panel, and a live JSON API — most of it
generated by the framework.

Reading time: ~30 minutes.

<!-- screenshot: final home page with two seeded links -->

## 1. Prerequisites

- Node.js 20 or newer (`node --version`).
- That's it. SQLite ships in-tree, no other services required.

> **Heads-up.** On Node < 20 the CLI's ES module features will fail on
> import. Upgrade before starting.

## 2. Scaffold the app

Run:

```bash
npm install -g git+https://github.com/foobarjs/foobarjs.git#v0.4.0
foobar new links-app
```

> **Note.** Installing from an HTTPS tarball is a stopgap until foobarjs is
> published to npm — it works without any SSH keys or `git` configured on
> the machine.

`foobar new` will prompt for your first admin user (name, email, password),
then write the skeleton, install dependencies, seed the schema, and create
that user. Answer the prompts, then:

```bash
cd links-app
foobar serve --dev
```

You should see a boot line and — because the API layer is auth-required
by default in v0.4.0 — a **surface report** listing every mounted API
route with its middleware chain:

```
foobarjs dev mode (watch + live-reload) — http://localhost:3000

[foobar] API surface (auth default: 'auth'):
  GET     /api/users              api.users.index    [auth]
  ...
```

`PUBLIC` (bold red on a TTY) marks any route without auth. Right now
your fresh scaffold has no public API routes — everything requires an
authenticated user. Section 8 shows how to opt a resource out.

Open <http://localhost:3000>. The default home page renders. Open
<http://localhost:3000/admin> and log in with the admin credentials you
supplied — the auto-admin already lists your `User` model.

Leave `foobar serve --dev` running for the rest of the tutorial; it watches
files and reloads on save.

## 3. Add the Link model

Generate it:

```bash
foobar g model link -f "title:string:required:minLength=2,url:string:required:url,clicks:number:unsigned:default=0"
```

Output:

```
✓ Created model: app/models/link.model.js
```

That produces exactly the model we want — the `-f` DSL accepts chainable
modifiers separated by colons, so no manual editing is needed:

<!-- app/models/link.model.js -->
```js
import { Model, Field } from 'foobarjs/orm'

class Link extends Model {
  static schema = {
    title: Field.string().required().minLength(2),
    url: Field.string().required().url(),
    clicks: Field.number().unsigned().default(0),
  }

  static timestamps = true
}

export default Link
```

Because `foobar serve --dev` is watching, the framework
restarts and syncs the schema for you — you'll see a `links` table
created in the console output. No `db:sync` command needed.

> **Heads-up.** Auto-sync is on because the scaffold hasn't generated a
> `database/migrations/` folder yet. The moment you run `foobar db:make`
> (to snapshot your first real migration), the boot switches to
> "migrations-only" mode and `foobar db:migrate` becomes the way you
> promote schema changes. That's the path you'd use in CI / production;
> the auto-sync is a dev-loop convenience.

## 4. See it in the admin panel

Open <http://localhost:3000/admin> again. There's now a **Links** entry in
the sidebar. Click it, hit **New**, add a couple of rows:

| title | url | clicks |
|-------|-----|--------|
| foobarjs docs | https://foobarjs.github.io/docs | 0 |
| Hacker News | https://news.ycombinator.com | 0 |

You wrote zero admin code. `foobarjs/admin` (registered in `config/app.js`)
discovers every model and mounts a CRUD screen automatically. When you want
to customize columns, filters, or actions, drop a file at
`app/admin/link.admin.js` — see [admin-panel.md](./admin-panel.md).

## 5. Create the controller

```bash
foobar g controller links
```

Replace the generated file with:

<!-- app/controllers/links.controller.js -->
```js
import { Controller, NotFoundError } from 'foobarjs/core'
import Link from '../models/link.model.js'

class LinksController extends Controller {
  static auth = false

  async index() {
    const links = await Link.orderBy('createdAt', 'desc').get()
    return this.render('links/index', { links })
  }

  async show() {
    const link = await Link.find(this.param('id'))
    if (!link) throw new NotFoundError('Link not found')
    return this.render('links/show', { link })
  }
}

export default LinksController
```

You don't need to touch `routes/web.js`. Any file matching
`app/controllers/<name>.controller.js` is auto-mounted as a REST resource
at `/<name>`. Only the actions you *define* get routes, so right now
`LinksController` exposes:

| Method | Path | Action |
|--------|------|--------|
| GET | `/links` | `index` |
| GET | `/links/:id` | `show` |

Confirm it:

```bash
foobar routes
```

`static withoutMiddleware = ['auth']` opts this controller out of the
global auth requirement so anonymous visitors can browse. The equivalent
per-route form is `router.get('/links', ...).public()`.

## 6. Write the views

Create the directory and two files:

<!-- app/views/links/index.jsx -->
```jsx
import App from '../layouts/App.jsx'

export default function LinksIndex({ links }) {
  return (
    <App title="Links">
      <section class="hero">
        <h1>Links</h1>
        <p class="tagline">A tiny bookmark manager.</p>
        <p><a href="/links/new" class="btn">Add a link</a></p>
      </section>

      {links.length === 0 ? (
        <p>No links yet.</p>
      ) : (
        <ul class="link-list">
          {links.map(link => (
            <li key={link.id}>
              <a href={`/links/${link.id}`}>{link.title}</a>
              <span class="muted"> · {link.clicks} clicks</span>
            </li>
          ))}
        </ul>
      )}
    </App>
  )
}
```

<!-- app/views/links/show.jsx -->
```jsx
import App from '../layouts/App.jsx'

export default function LinkShow({ link }) {
  return (
    <App title={link.title}>
      <section class="hero">
        <h1>{link.title}</h1>
        <p><a href={link.url} target="_blank" rel="noopener">{link.url}</a></p>
        <p class="muted">{link.clicks} clicks</p>
        <p><a href="/links">← Back to all links</a></p>
      </section>
    </App>
  )
}
```

Reload <http://localhost:3000/links> — the two links you added via the
admin panel show up. Click one; the detail page renders.

> **Heads-up.** foobarjs's JSX uses `class=""` (the HTML attribute), not
> React's `className`. It's server-rendered to a string, not hydrated.

## 7. Add a create form

Create the validator first:

```bash
foobar g validator create-link
```

Replace it with:

<!-- app/validators/create-link.validator.js -->
```js
import { FormRequest } from 'foobarjs/validation'
import { Field } from 'foobarjs/orm'

export default class CreateLinkValidator extends FormRequest {
  rules() {
    return {
      title: Field.string().required().minLength(2).trim(),
      url: Field.string().required().url().trim(),
    }
  }
}
```

Now extend the controller with `new()` and `store()`:

<!-- app/controllers/links.controller.js -->
```js
import { Controller, NotFoundError } from 'foobarjs/core'
import Link from '../models/link.model.js'
import CreateLinkValidator from '../validators/create-link.validator.js'

class LinksController extends Controller {
  static auth = false

  async index() {
    const links = await Link.orderBy('createdAt', 'desc').get()
    return this.render('links/index', { links })
  }

  async show() {
    const link = await Link.find(this.param('id'))
    if (!link) throw new NotFoundError('Link not found')
    return this.render('links/show', { link })
  }

  async new() {
    return this.render('links/new', {})
  }

  async store() {
    const request = await this.validateOrBack(CreateLinkValidator)
    const link = await Link.create(request.validated())
    this.flash('success', `Saved "${link.title}".`)
    return this.redirect(`/links/${link.id}`)
  }
}

export default LinksController
```

`foobar routes` will now show two more entries:

| Method | Path | Action |
|--------|------|--------|
| GET | `/links/new` | `new` |
| POST | `/links` | `store` |

Add the form view:

<!-- app/views/links/new.jsx -->
```jsx
import App from '../layouts/App.jsx'
import { useView } from 'foobarjs/jsx'

export default function LinkNew() {
  const { errors, old } = useView()
  return (
    <App title="New link">
      <section class="hero">
        <h1>New link</h1>

        <form method="POST" action="/links">
          <div class="form-group">
            <label for="title">Title</label>
            <input
              id="title"
              name="title"
              value={old('title') || ''}
              class={errors?.title ? 'is-invalid' : ''}
            />
            {errors?.title && <div class="invalid-feedback">{errors.title[0]}</div>}
          </div>

          <div class="form-group">
            <label for="url">URL</label>
            <input
              id="url"
              name="url"
              type="url"
              placeholder="https://…"
              value={old('url') || ''}
              class={errors?.url ? 'is-invalid' : ''}
            />
            {errors?.url && <div class="invalid-feedback">{errors.url[0]}</div>}
          </div>

          <button type="submit" class="btn">Save link</button>
          {' '}
          <a href="/links">Cancel</a>
        </form>
      </section>
    </App>
  )
}
```

Visit <http://localhost:3000/links/new>, submit an empty form, and watch
the validation errors render. Submit a real URL — you get redirected to the
show page, `clicks` defaulted to `0`, `createdAt` set.

`useView()` pulls per-request bag data set by `withErrors` / `withInput`.
`old(name)` returns the previous submission so the user isn't retyping
after a validation failure.

> **CSRF and curl.** `POST /links` above is a WEB route (registered in
> `routes/web.js` / auto-mounted from the controller), so CSRF's
> Origin/Referer check applies. To POST to a web route from curl, include
> `-H "Origin: http://localhost:3000"`. API routes (`routes/api.js`,
> `/api/*`) have CSRF **off** by default (since v0.3.2), so `curl -X POST
> http://localhost:3000/api/links -d '{...}'` just works without any
> Origin header.

## 8. Use the API

You already have one. Because `foobarjs/api` is registered in
`config/app.js` and `Link` is a discovered model, five REST endpoints
were auto-mounted at boot — you saw them in the API surface report in §2.

| Method | Path |
|--------|------|
| GET | `/api/links` |
| GET | `/api/links/:id` |
| POST | `/api/links` |
| PUT | `/api/links/:id` |
| DELETE | `/api/links/:id` |

**They all require authentication by default** in v0.4.0. A bare curl
gets 401:

```bash
curl -s http://localhost:3000/api/links
# → {"error":"Unauthenticated","status":401,"requestId":"..."}
```

For a public browse-anyone-can API, mark the resource `.public()`.
Create one file:

<!-- app/api/link.api.js -->
```js
import { Api } from 'foobarjs/api'
import Link from '../models/link.model.js'

export default Api.resource(Link).public()
```

Save. Watch reloads. The API surface report reprints — now `PUBLIC`
next to every `api.links.*` route. Try again:

```bash
curl -s http://localhost:3000/api/links | jq
```

Output:

```json
{
  "data": [
    { "id": 2, "title": "Hacker News", "url": "https://news.ycombinator.com", "clicks": 0, "createdAt": "...", "updatedAt": "..." },
    { "id": 1, "title": "foobarjs docs", "url": "https://foobarjs.github.io/docs", "clicks": 0, "createdAt": "...", "updatedAt": "..." }
  ],
  "meta": { "currentPage": 1, "perPage": 15, "total": 2, "lastPage": 1 }
}
```

For per-verb control — e.g. reads public, writes still auth-required:

```js
export default Api.resource(Link).public('index', 'show')
// GET /api/links + GET /api/links/:id are public
// POST/PUT/DELETE stay auth-required
```

To flip the *global* default so every model auto-mounts public (v0.3.x
behavior), set `auth.default: 'public'` in `config/auth.js`. Do this
only if you know none of your models expose sensitive data — the point
of the v0.4.0 default is that a stray `PasswordReset` model doesn't
silently become a public JSON dump.

Interactive docs are mounted at <http://localhost:3000/api/docs>.

## 9. What just happened

Four conventions did most of the work:

1. **Model discovery.** `app/models/*.model.js` is scanned at boot; every
   class becomes a booted model with a table.
2. **Controller auto-mount.** `app/controllers/links.controller.js`
   registered `/links` routes for every action you defined — no
   `routes/web.js` edit needed.
3. **Auto-admin.** `foobarjs/admin` reads the model schema and mounts a
   CRUD screen at `/admin/links`.
4. **Auto-API.** `foobarjs/api` mounts a `/api/links` REST resource with
   pagination and a `{ data, meta }` envelope.

The parts you *did* write — the model schema, two controller actions, two
views, a validator, one form — are the parts unique to your app.
